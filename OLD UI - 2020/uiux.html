Appalogue - Ruby on Rails App Directory

Appalogue is a basic app catalogue web application created with Ruby on Rails as part of a group project for CSE 201: Intro to Software Engineering. The goal of this course was to learn about the Agile Software Development process by build a web application that people could conceivably use.

When first visiting the site, users can see a list of all apps available on Appalogue. However, to interact with some of Appalogue's features, users must first create an account using an email address. Once logged in, users have the ability to leave one review per app by viewing the app's info and leaving a review (a star rating and a comment). If the review is offensive or of poor quality, a moderator can remove the review. When logged in, users can also submit app requests to be added to Appalogue. These requests can then be approved or denied by an admin. If approved, the app is then shown to all users on the Appalogue home screen. Users may also search for apps in two ways: using the search bar to search by app name, or using advanced search (in the bottom navigation area) to search by other fields, like developer or price. Lastly, there is a Help / FAQ page in the bottom navbar, if the user is confused or needs additional help.

For this project, I was in charge of the user interface design, app view / creation / deletion, user login, user permissions (admin, moderator, etc.), and star reviews. The UI was created from scratch using HTML / CSS, while features like user login and star ratings were handled by Ruby gems (Devise and jQuery.raty, a jQuery plugin).

Virtual Education - Mobile Website


Virtual Education is a simple website that was made for IMS 222: Introduction to Interaction Design and Development. The focus of this class was on designing and creating effective web interfaces with HTML, CSS, and JavaScript for various devices; this project was for mobile phones. While designing the website, I incorporated many mobile-friendly design elements, including a button to immediately return to the top of a scrolled page and placing the navbar at the bottom of the screen.

For theis assignment, the website's content was supposed to be about creating a business for "an industry which needed disrupting". Thus, I chose to create a website for a educational VR company, which provided "serious games" (educational and training games) to companies and schools.

Business Assessment Association


The Business Assessment Association is a satirical website that was made as part of a group project for IMS 257: Web Interaction Programming (now IMS 322: Intermediate Interaction Design and Development. This class was a continuation of IMS 222, diving deeper into techniques for creating HTML, CSS, and JavaScript interfaces for websites.This assignment revolved around creating a business website that could conceivably be used in the professional world. One of our group members suggested we go over-the-top and just make the whole thing full of generic business buzzwords, and the rest is history.

Each group member was in charge of creating a webpage for the site - my page is the "Estimates" page, which uses four number inputs to generate a random number (and a "Kaching!" sound). I also was in charge of assembling the overall website, meaning that I had to combine each person's pages into a coherent website.

MuseScore 3.0 Interface Evaluation and Redesign

In IMS 315: User Experience Research, we were tasked with a semester-long interface evaluation and re-design for a website / software of our choosing. I chose to evaulate MuseScore, which is open-source music notation software that I frequently use.

To start, I first drafted a usability test for evaluating the effectiveness of MuseScore's UI and UX. I then ran this test on five sepearate participants, selected as people who are part of MuseScore's intended audience. Using these results, I drafted a usability report, which can be viewed below. Additionally, I did some card sort test to figure out the best names / groupings for the menu in MuseScore.

I then used the results from the report to create wireframes and mockups (shown above), which reflect the findings in the report and illustrate potential ways to improve MuseScore's user interface and experience.
