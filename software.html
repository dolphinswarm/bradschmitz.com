OpenGL Graphics

OpenGL Ray Tracer


OpenGL Ray Tracer is a graphics program that was created for CSE 287 (now 386): Introduction to Computer Graphics and Games. For this project, our goal was to use the open-source C++ graphics library OpenGL to create a basic ray tracer for rendering 3D scenes. Some of the features in this ray-tracer include low-level implementations of ambient, directional, positional, and spot lights, shadows created using "shadow feelers", inter-object reflections between objects, multiple camera angles, and anti-aliasing. This course was more concerned with the implementations instead of the execution of said features; thus, the program runs on the CPU instead of the GPU and is slower than usual.

Controls:
• B: Toggle fullscreen
• Esc: Quit application
• 1-4: Set Recursion Depth
• A: Toggle ambient light
• D: Toggle directional light
• P: Toggle positional light
• S: Toggle spot light
• F-H: Change camera position
• Z: Toggle antialiasing

OpenGL Vertex Graphics Engine


Vertex-Based Graphics Engine is a graphics program that was created for CSE 287 (now 386): Introduction to Computer Graphics and Games. This project was meant to show off vertex-based graphics pipeline, which is used in most 3D game engines. Some of the features of this project include low-level implementations of basic animaiton, per vertex / per fragment shading, perspective / orthographic projections, a free-moving camera, object clipping, and backface culling. This course was more concerned with the implementations instead of the execution of said features; thus, the program runs on the CPU instead of the GPU and is slower than usual.

The application is controlled through a right-click window.

OpenGL Game Engine

OpenGL Game Engine is vertex-based rendering and game engine created during CSE 387 (now 489): Game Engine Design. This class was a continuation of CSE 287 (now 386), and discussed the techniques and algorithms behind 3D graphics generation in real-time engines, such as those used in games. The first part of the class revolved around learning how OpenGL worked and how to effectively use it in graphics applications (such as buffers, shaders, uniform blocks, etc.), while the second half used that information to construct a basic game engine. Included in this game engine were texture mapping, model loading (using ASSIMP), cameras, game objects, components, scene graphs and scene graph transformation, sound (using FMOD), physics (using Bullet), and quaternions.

In the summer and fall of 2020, I also helped the course's professor (Dr. Eric Bachmann) in developing some extra features for the project, including mouse picking, cubemaps, 2D text (using FreeType), and more.

Virtual / Augmented Reality

AR Looper

AR Looper is an augmented reality application, developed using Unity and Vuforia. This application allows people to point their phone at a printed-out piece of paper and make their own simple music loops. Users can select from five instrument sets, with five instruments per set (25 instruments total). Users can also add a lowpass, highpass, and reverb filters to their remix.

You may be asking to yourself, "why do you need AR for this?" Initially, this project was conceived to be a mobile launchpad / keyboard for on-the-go music composition and arrangement, but we felt that AR technology wasn't good enough to achieve our desired effect. We found that Vuforia's buttons were a lot less responsive than initially planned, so we had to scale our project down to be a loop machine. Maybe AR technology will be good enough one day to allow for a cheap and mobile AR launchpad?

X-Ray Locker Viewer

X-Ray Locker Viewer is a proof-of-concept joke augmented reality application, developed using Unity and AF Foundation. The application is simple - line up the crosshairs with a locker, then tap the screen to see what's in the locker! Total, there are 7 different lockers that a user can see, most being Halloween-themed (as the app was made around that time). The app also uses an occlusion shader to hide the locker from any angle that isn't the front, giving the appearance that the locker was "opened".

Parkinson's VR Training

Parkinson's VR Training is a virtual reality training simulation that was developed as part of a research project. This project was a collaboration between myself, Dr. Eric Hogsdon, a 3D modeling / animation student, and a nursing professor. Through our combined talents, we developed a simulation that nursing students can use to mimic handling a Parkinson's patient. The nursing professor intends to use this program to conduct a study on the effectiveness of virtual reality on training in the medical field.

This project is powered through a scripting event system, and features many interactive events for user immersion - giving medicine through a syringe, performing auscultation, talking out loud to other characters, and more. With the help of this study, we will hopefully be able to determine the effectiveness of VR in training all kinds of professions.

Statistical Projects

The Simpsons in Statistics

The Simpsons in Statistics is an R Shiny dashboard developed as part of STA/IMS 404: Advanced Data Visualization. This app uses information from TMDb and data.world to see how the popular American animated sitcom The Simpsons has changed in its 30+ year run. Included are analyzations and visualizations of ratings over time, number of guest stars, and popularity/frequency of guest stars, directors, and writers. Users can also select which seasons they would like to display to help compare seasons.

This app helps to visualize a lot of aspects of The Simpsons, including its rise and fall in ratings (particularly the "golden years"), how certain individuals contributed to the show (for example, which people were the most popular and/or frequent guest stars?), and how the show has generally changed over the years.

Health Data Analysis via SAS

Health Data Analysis via SAS is the final project for STA 402: Statistical Programming. In this project, we had to download a large dataset (over 1 GB!), convert it into a SAS format, perform data cleaning, then analyze the results using visualization tools in SAS. For my project, I downloaded information from the US CDC's Behavioral Risk Factor Surveillance System survey, which polls over 100,000 people about their health behaviors and risk factors. Using this, I checked to see if a person's diet (including fruit and veggie consumption, sodium consumption, etc.) would be a significant predictor of their physical and mental health. The results of my report and my code for cleaning and visualizing the data are included.
