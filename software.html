OpenGL Graphics

OpenGL Ray Tracer


OpenGL Ray Tracer is a graphics program that was created for CSE 287: Introduction to Computer Graphics and Games. For this project, our goal was to use the open-source C++ graphics library OpenGL to create a basic ray tracer for rendering 3D scenes. Some of the features in this ray-tracer include low-level implementations of ambient, directional, positional, and spot lights, shadows created using "shadow feelers", inter-object reflections between objects, multiple camera angles, and anti-aliasing. This course was more concerned with the implementations instead of the execution of said features; thus, the program runs on the CPU instead of the GPU and is slower than usual.

Controls:
• B: Toggle fullscreen
• Esc: Quit application
• 1-4: Set Recursion Depth
• A: Toggle ambient light
• D: Toggle directional light
• P: Toggle positional light
• S: Toggle spot light
• F-H: Change camera position
• Z: Toggle antialiasing

OpenGL Vertex Graphics Engine


Vertex-Based Graphics Engine is a graphics program that was created for CSE 287: Introduction to Computer Graphics and Games (now CSE 386). This project was meant to show off vertex-based graphics pipeline, which is used in most 3D game engines. Some of the features of this project include low-level implementations of basic animaiton, per vertex / per fragment shading, perspective / orthographic projections, a free-moving camera, object clipping, and backface culling. This course was more concerned with the implementations instead of the execution of said features; thus, the program runs on the CPU instead of the GPU and is slower than usual.

The application is controlled through a right-click window.

OpenGL Game Engine

OpenGL Game Engine is vertex-based rendering and game engine created during CSE 387: Game Engine Design. This class was a continuation of CSE 287 (now 386), and discussed the techniques and algorithms behind 3D graphics generation in real-time engines, such as those used in games. The first part of the class revolved around learning how OpenGL worked and how to effectively use it in graphics applications (such as buffers, shaders, uniform blocks, etc.), while the second half used that information to construct a basic game engine. Included in this game engine were texture mapping, model loading (using ASSIMP), cameras, game objects, components, scene graphs and scene graph transformation, sound (using FMOD), physics (using Bullet), and Quaternions.

Our final for this class was using the game engine we created to create a rudimentary game. I chose to make a pretty basic Rocket League clone, with various objects for score and a primitive AI tank enemy.

Virtual / Augmented Reality

AR Looper

AR Looper is an augmented reality application, developed using Unity and Vuforia. This application allows people to point their phone at a printed-out piece of paper and make their own simple music loops. Users can select from five instrument sets, with five instruments per set (25 instruments total). Users can also add a lowpass, highpass, and reverb filters to their remix.

You may be asking to yourself, "why do you need AR for this?" Initially, this project was conceived to be a mobile launchpad / keyboard for on-the-go music composition and arrangement, but we felt that AR technology wasn't good enough to achieve our desired effect. We found that Vuforia's buttons were a lot less responsive than initially planned, so we had to scale our project down to be a loop machine. Maybe AR technology will be good enough one day to allow for a cheap and mobile AR launchpad?
