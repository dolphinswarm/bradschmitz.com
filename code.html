Virtual / Augmented Reality

VRchery

  • Year Created: 2019
  • Software / Languages Used: Unity Game Engine, OculusVR, C#
  • Overview: A simple archery demo in virtual reality. The player has 10 attempts to hit a target ~25 meters away, and receives a score based on how well they did. Included in the project are major VR concepts like haptic feedback and realistic bow pulling and firing.

Rowan Hall Scavenger Hunt

  • Year Created: 2019
  • Software / Languages Used: Unity Game Engine, AR Foundation, C#
  • Overview: This app was part of a class effort to create a series of applications commemorating the 50th Anniversary of the Rowan Hall Protests at Miami Univerity. Each group was assigned a certain subtopic, with ours being given BAM (Black Action Movement) and BSAA (Black Student Action Association). This AR application is meant to be played in the Shade Family Room in the Armstrong Student Center at Miami University. Users first open up the app, then move to the center of the room. Once there, the player has to hunt for megaphones placed in various locations around the room. Once found, the megaphones read some information about BAM and BSAA. Also featured in the application are a table with newspaper clippings from the time and some protestors.

AR Looper

X-Ray Locker Viewer

  • Year Created: 2019
  • Software / Languages Used: Unity Game Engine, AR Foundation, C#
  • Overview: A proof-of-concept joke augmented reality application. To use the app, simply line up the crosshairs with a locker, then tap the screen to see what's in the locker! Computer Graphics

OpenGL Ray Tracer

  • Year Created: 2019
  • Software / Languages Used: OpenGL, C++
  • Overview: A computer graphics program created to showcase the fundamentals of ray traying 3D scenes. Some of the features in this ray tracer include low-level implementations of ambient, directional, positional, and spot lights, shadows created using "shadow feelers", inter-object reflections between objects, multiple camera angles, and anti-aliasing. Due to the implementation of this program on the CPU instead of the GPU, the ray tracer runs slower than a typical ray tracer.
  • Controls:
    • B: Toggle fullscreen
    • Esc: Quit application
    • 1-4: Set Recursion Depth
    • A: Toggle ambient light
    • D: Toggle directional light
    • P: Toggle positional light
    • S: Toggle spot light
    • F-H: Change camera position
    • Z: Toggle antialiasing

OpenGL Vertex Graphics Engine


  • Year Created: 2019
  • Software / Languages Used: OpenGL, C++
  • Overview: A computer graphics program created to showcase a vertex-based graphics pipeline. Some of the features of this project include low-level implementations of basic animaiton, per vertex / per fragment shading, perspective / orthographic projections, a free-moving camera, object clipping, and backface culling. Due to the implementation of this program on the CPU instead of the GPU, the program runs slower than a typical render engine.
  • Controls: The application is controlled through a right-click window.

Vertex-Based Graphics Engine is a graphics program that was created for CSE 287 (now 386): Introduction to Computer Graphics and Games. This project was meant to show off vertex-based graphics pipeline, which is used in most 3D game engines. This course was more concerned with the implementations instead of the execution of said features; thus, the program runs on the CPU instead of the GPU and is slower than usual.

The application is controlled through a right-click window.

Appalogue

  • Year Created: 2019
  • Software / Languages Used: Ruby on Rails
  • Overview: A basic app catalogue web application, developed as a course project for a class. For this project, I was in charge of the user interface design, app view / creation / deletion, user login, user permissions (admin, moderator, etc.), and star reviews. The UI was created from scratch using HTML / CSS, while features like user login and star ratings were handled by Ruby gems (Devise and jQuery.raty, a jQuery plugin).

    When first visiting the site, users can see a list of all apps available on Appalogue. However, to interact with some of Appalogue's features, users must first create an account using an email address. Once logged in, users have the ability to leave one review per app by viewing the app's info and leaving a review (a star rating and a comment). If the review is offensive or of poor quality, a moderator can remove the review. When logged in, users can also submit app requests to be added to Appalogue. These requests can then be approved or denied by an admin. If approved, the app is then shown to all users on the Appalogue home screen. Users may also search for apps in two ways: using the search bar to search by app name, or using advanced search (in the bottom navigation area) to search by other fields, like developer or price. Lastly, there is a Help / FAQ page in the bottom navbar, if the user is confused or needs additional help.

The Simpsons in Statistics

  • Year Created: 2020
  • Software / Languages Used: R, R Shiny
  • Overview: An R Shiny dashboard developed as a course priject. This app uses information from TMDb and data.world to see how the popular American animated sitcom The Simpsons has changed in its 30+ year run. Included are analyzations and visualizations of ratings over time, number of guest stars, and popularity/frequency of guest stars, directors, and writers. Users can also select which seasons they would like to display to help compare seasons.
